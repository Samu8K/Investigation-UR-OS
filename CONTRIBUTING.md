# Contribuir a Investigation UR-OS

Gracias por tu interés en contribuir a este proyecto. Estas son las pautas para mantener el trabajo organizado y profesional.

## Antes de comenzar

- Revisa la documentación en `README.md`.
- Comprueba si existe un issue abierto para tu aporte.
- Crea un issue si quieres proponer una mejora o reportar un problema.

## Flujo de trabajo recomendado

1. Crea una rama nueva a partir de `main`:
   ```bash
   git checkout -b feature/nombre-descriptivo
   ```
2. Escribe código claro y mantenible.
3. Añade comentarios útiles cuando sea necesario.
4. Agrega pruebas en `tests/`.
5. Haz commits pequeños y descriptivos.
6. Envía un pull request con una descripción clara del cambio.

## Formato de commits

Usa mensajes de commit que expliquen el cambio en una línea, por ejemplo:

- `feat: agregar módulo de análisis de datos`
- `fix: corregir validación de entrada`
- `docs: actualizar documentación de instalación`

## Revisión de pull requests

- Asegúrate de que el código es consistente con el estilo del proyecto.
- Verifica que todas las pruebas relevantes pasen.
- Explica en el PR por qué el cambio es necesario.

## Buenas prácticas

- Mantén el repositorio limpio de archivos temporales.
- Usa el formato y estilo adecuado para el lenguaje que se implemente.
- Documenta cualquier dependencia o configuración especial.
